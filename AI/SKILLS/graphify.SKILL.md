\# Graphify Usage Skill



\## Purpose

Use graphify knowledge graph to understand codebase structure before making changes.



\## Rules

\- Always read graphify-out/GRAPH\_REPORT.md before major changes

\- Prefer graph traversal over guessing file structure

\- Do not scan entire repo blindly



\## When to use

\- Understanding architecture

\- Finding related files

\- Planning changes across modules
